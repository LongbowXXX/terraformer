# Knowledge Templates

This directory categorizes templates based on their usage in the [Standard Workflow](./workflows/workflow.md).

## Directory Structure

### 1. [Issue Templates](./templates/issues/)

Templates for **Process Management** (GitHub Issues).
We use a **Story (Parent)** - **Task (Child)** hierarchy.

#### Story Templates (Parent)

- `story_feature.md`: Feature Addition/Change.
- `story_release.md`: Release Process.
- `story_bug_fix.md`: Bug Fixes.

#### Task Templates (Child)

- `task_requirement.md`: Requirement Definition.
- `task_design.md`: Design Discussion.
- `task_implementation.md`: Implementation & PR.
- `task_test_sanity.md`: Sanity Test Reporting.
- `task_test_functional.md`: Functional Test Reporting.
- `task_exit_criteria.md`: Final Exit Criteria Check.

### 2. Product Templates

Templates for persistent artifacts and agent outputs.

#### [Artifacts](./templates/artifacts/)

Persistent files in the repository (`docs/`).

- **Location**: `knowledge/templates/artifacts/`
- **Examples**:
  - `specification.template.md`
  - `design.template.md`

#### [Agents](./templates/agents/)

Transient reports or plans generated by agents.

- **Location**: `knowledge/templates/agents/`
- **Examples**:
  - `bug_fix_plan.template.md`: Generated by @Debugger.
  - `review_report.template.md`: Generated by @QualityGuard.

### 3. [Guidelines](./guidelines/)

Guides and rules that define **"How we work"** in this project.
Refer to these documents for standards on quality, reviewing, and AI collaboration.

- **Location**: `knowledge/guidelines/`
- **Key Documents**:
  - `adoption-guide.md`: How to introduce Terraformer to a project.
  - `debugging.md`: The scientific method for debugging.
  - `pr-creation-guidelines.md`: Standards for submitting Pull Requests.
  - `software-review.md`: Checking points for code review.
  - `specification-guidelines.md`: How to write unambiguous specifications.
